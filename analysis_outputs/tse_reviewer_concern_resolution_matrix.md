the remaining evidence gaps. It is intentionally claim-conservative: an item is
marked addressed only when the current paper text and audited outputs support it.

## Current Reporting Recommendation

1. Main tables should report pass@1, Avg./Med. AED, and Avg./Med. CCR.
2. Use complete Base/SFT/PAFT blocks for Qwen3-8B, Qwen2.5-Coder-7B, and OpenCoder-8B; use DS-Coder-6.7B as the ablation-linked reference; use Qwen2.5-Coder-14B as the larger-backbone diagnostic.
3. Keep frontier API rows as diagnostic references only.
4. Keep QuixBugs-Python as a cross-language sanity check only.
5. Keep the Multi-SWE Java smoke10 result as a generator-level complex-benchmark diagnostic only; do not present it as official harness pass/fail.
6. Do not report pass@5/pass@10 in the main TSE claims unless a reviewer explicitly asks; they remain internal robustness notes.
7. Do not claim system-level APR state of the art without a generator-swap or full pipeline comparison.
